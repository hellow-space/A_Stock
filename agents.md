# Agent 协作工作流程

## 基本原则

1. **本地优先**：所有修改先在本地文件完成
2. **同步先行**：修改前必须先拉取仓库最新内容
3. **及时推送**：修改后立即提交并推送到 GitHub
4. **保持一致**：确保本地与仓库内容始终同步

---

## 工作流程

### 工作前
1. 执行 `git pull origin main` 拉取最新内容
2. 读取 `README.md` 了解项目当前状态
3. 检查变更记录，确认上次修改内容

### 工作中
1. 在本地编辑文件
2. 修改 `README.md` 的变更记录（如有需要）
3. 确保文件内容准确完整

### 工作后
1. 执行 `git add .`
2. 执行 `git commit -m "描述修改内容"`
3. 执行 `git push origin main`
4. 确认推送成功

---

## 冲突处理

如果 `git pull` 时出现冲突：

1. **不要强行推送**
2. 查看冲突文件：`git status`
3. 手动编辑冲突文件，保留正确内容
4. 重新提交：`git add . && git commit -m "解决冲突"`
5. 再推送：`git push origin main`

---

## 文件规范

1. **文档格式**：Markdown (.md)
2. **编码**：UTF-8
3. **命名**：英文小写，单词间用连字符 `-`
4. **目录**：
   - 宏观研究 → `docs/macro/`
   - 行业研究 → `docs/industry/`
   - 公司研究 → `docs/company/`
   - 策略研究 → `docs/strategy/`

---

## 检查清单

每次工作完成后确认：

- [ ] 本地修改已完成
- [ ] README 变更记录已更新（如有修改）
- [ ] 已执行 `git add .`
- [ ] 已执行 `git commit`
- [ ] 已执行 `git push origin main`
- [ ] 推送成功无报错
